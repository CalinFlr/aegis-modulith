# Workflow: Create Spec

## Purpose

Create a durable spec for meaningful work.

## Steps

1. Read `AGENTS.md`.
2. Read `OpenQuestions.md`.
3. Copy `specs/_template`.
4. Fill in problem, goals, non-goals, requirements, tasks, risks, and acceptance.
5. Promote cross-cutting questions to root `OpenQuestions.md`.

## Required validation

```bash
npm run check:specs
```

## Human approval

Required for high or critical risk specs.
