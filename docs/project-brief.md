# Project Brief: Aegis.Modulith

## Elevator pitch

`Aegis.Modulith` is a production-minded .NET modular monolith template system with CQRS-lite, vertical slices, PostgreSQL, Aspire, OpenTelemetry, optional MediatR, and enterprise AI development guardrails.

## Target users

- .NET developers starting personal APIs.
- Senior developers who want a strong starter without ceremony.
- Teams standardizing modular monolith practices.
- Open-source users looking for a modern alternative to generic Clean Architecture templates.
- AI-assisted development teams that need guardrails, workflows, skills, policies, and validation.

## Differentiator

Many .NET starter templates include Clean Architecture, MediatR, EF Core, Swagger, and tests. `Aegis.Modulith` differentiates with:

- Modular monolith first, not microservices by default.
- CQRS-lite, not event sourcing by default.
- Vertical slices inside modules.
- PostgreSQL schema per module.
- Optional MediatR, core dispatcher by default.
- `dotnet new` profiles: core, pro, advanced.
- Item templates for modules, slices, events, workers, docs.
- Enterprise AI development system: AGENTS.md, skills, workflows, guardrails, policies, evals.
- Node `.mjs` guardrail runner instead of duplicated shell scripts.
- Strong docs and ADRs.

## Product tagline

A pragmatic .NET modular monolith template with CQRS, vertical slices, PostgreSQL, Aspire, OpenTelemetry, and enterprise AI development guardrails.
