# Name Decision

## Chosen name

`Aegis.Modulith`

## Why

`Aegis` means shield or protection. It reflects the project's strongest differentiator: enterprise AI development with guardrails, policies, workflows, and verification. `Modulith` keeps the modular monolith identity visible for search and positioning.

## Proposed identifiers

- GitHub repository: `aegis-modulith`
- NuGet package: `Aegis.Modulith.Templates`
- Root namespace: `Aegis.Modulith`
- Main template short name: `aegis-modulith`
- Module template: `aegis-module`
- Slice template: `aegis-slice`
- Event template: `aegis-event`
- Worker template: `aegis-worker`

## Alternatives considered

| Name | Pros | Cons |
|---|---|---|
| `Pragma.Modulith` | Pragmatic, clean, technical | Weaker guardrails/security identity |
| `Forge.Modulith` | Strong builder/toolkit metaphor | Less specific to protection/AI safety |
| `Sentinel.Modulith` | Strong guardrails/security metaphor | Sounds more like monitoring/security-only product |
| `Keystone.Modulith` | Stable architecture metaphor | More generic, less memorable |
| `Axiom.Modulith` | Architecture/principles metaphor | Less clear for template/tooling |

## Availability note

Before publishing to NuGet or creating the public repository, verify current name availability on NuGet and GitHub. Do not assume availability from this specification alone.
